# ItemTracker - Technical Documentation

## 📋 Project Overview

**ItemTracker** is a professional inventory management system built with vanilla JavaScript, HTML5, and CSS3. It's a single-page application with zero backend requirements.

---

## 🚀 Getting Started

### Prerequisites
- Modern web browser (Chrome 60+, Firefox 55+, Safari 12+, Edge 79+)
- No backend server required for basic operation
- Python or any HTTP server for development

### Running Locally

**Option 1: Direct File**
```bash
# Just open the file
open index.html

# Or in browser
file:///path/to/index.html
```

**Option 2: Python HTTP Server** (Recommended)
```bash
# Navigate to project directory
cd path/to/ItemTracker-main

# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000

# Then open: http://localhost:8000
```

**Option 3: Other HTTP Servers**
```bash
# Node (with http-server)
npx http-server

# Ruby
ruby -run -ehttpd . -p8000

# PHP
php -S localhost:8000
```

## 📖 Code Architecture

### app.js - Data Management
**Responsibilities:**
- Global data object (`data`) containing all inventory
- CRUD operations for parties, months, sizes, entries
- Calculations and aggregations
- localStorage persistence
- UI rendering for all screens

**Key Functions:**
```javascript
load()              // Load from localStorage
save()              // Save to localStorage
addParty(name)      // Create new party
getParty(id)        // Retrieve party
partyTotal(party)   // Calculate party stats
allTotal(data)      // Calculate grand totals
```

**Data Structure:**
```javascript
{
  parties: [
    {
      id: 'party1',
      name: 'Shubham',
      months: [
        {
          id: 'month1',
          name: 'January',
          sizes: [
            {
              id: 'size1',
              name: 'Small',
              entries: [
                { pcs: 100, wg: 500 },
                { pcs: 150, wg: 750 }
              ]
            }
          ]
        }
      ]
    }
  ]
}
```

## 📊 Export Architecture

### Excel Export Flow
```
data → Format as Array → XLSX.utils.aoa_to_sheet()
    → XLSX.utils.book_append_sheet() → XLSX.writeFile()
```

### PDF Export Flow
```
HTML String → html2pdf().set() → .from(html) 
    → .save() → Download PDF
```

### Export Optimization
- Direct HTML string to PDF (faster than DOM)
- requestAnimationFrame for non-blocking
- Error handling with toast notifications
- Async/await pattern for reliability

## 🔄 Data Flow

### Add Entry Flow
```
User Click → openEntryModal() → Show Form
    → User Submits → entryData {pcs, wg}
    → saveEntry() → Update data object
    → save() → localStorage
    → render() → Update UI
    → toast() → Notify user
```

### Customize Fields Flow
```
User Click → openCustomizationModal()
    → Load current fields from localStorage
    → User Changes → saveCustomFields()
    → localStorage.setItem()
    → location.reload()
    → All UI uses new field names
```

### Export Flow
```
User Click → openModal('m-export')
    → Choose export type
    → Gather data → Format for export
    → Generate file (Excel/PDF)
    → requestAnimationFrame() → avoid blocking
    → Download → toast notification
```

## 🐛 Debugging

### Console Logging
```javascript
// Enable debug mode
window.DEBUG = true;

// In any file:
if (window.DEBUG) console.log('Debug info:', data);
```

### Browser DevTools
1. **F12** - Open DevTools
2. **Console** - Check for errors
3. **Application** - View localStorage
4. **Network** - Check library loads
5. **Performance** - Profile app

### Common Issues & Solutions

**Issue: Data not saving**
- Check: `localStorage.getItem('itemtracker_data')`
- Solution: Browser storage not disabled

**Issue: Libraries not loading**
- Check: Network tab for jsDelivr CDN
- Solution: Check internet, try fallback CDN

**Issue: Charts not rendering**
- Check: `Chart` object in console
- Solution: Reload page, check Chart.js CDN

**Issue: PDF hangs**
- Check: HTML size with `html.length`
- Solution: Use party/size export instead
